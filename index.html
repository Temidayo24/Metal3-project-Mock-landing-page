Metal3

Bare Metal Host provisioning for Kubernetes

Metal3 (Metal Kubed)

Metal3 is a Kubernetes based project for managing bare metal machines.It is an infrastructure provider for the cluster API (K8s SIG Life-Cycle). It manages your underlying infrastructure through Kubernetes APIs, which allows you to install and scale Kubernetes; avoids the need for an external dependency by running inside your cluster; integrates with the machine API from SIG Cluster-Lifecycle, which allows your bare metal cluster to manage its own growth, similar to cloud-based clusters.

Metal3 is a kubernetes application and is therefore managed through Kubernetes interfaces.

Why Metal3?

Automated Cleaning

When enabled, Metal3 cleans host disks when node is provisioned for the first time and on every time deprovisioned.

Automatic secure boot

Install and securely boot an Operating System (OS) of your choice which protects the host from loading malicious code in the boot process before loading the provisioned Operating System.

Detached Annotation

Deploy Kubernetes and upgrade Kubernetes or the OS of your clusters with a non-disruptive rolling strategy.

Remediation

Automatically remediate failed nodes by rebooting them and removing them from the cluster if necessary.

Metal3 Components
Baremetal Operator (BMO)

The Baremetal Operator is a Kubernetes controller for managing BareMetal Hosts (BMH) resources. It assumes authority over Kubernetes Nodes, manages and enforces the desired state defined in a BareMetal Hosts which correspond to the Nodes Ports. And this is done using Ironic.

Cluster-api-provider-Metal3 (CAPM3)

Cluster-api-provider-Metal3 (CAPM3) is a declarative/self management API for Kubernetes used to create, configure and manage clusters. It provides infrastructure for the Kubernetes Cluster API and enables users to install clusters on BareMetal Hosts.

Ironic

Ironic is the hidden engine that operates to manage bare metal hosts behind native Kubernetes API. It automates the provisioning and managing of bare metal machines. It is API-driven and includes a full set of RESTful APIs. Its features include bare metal “nodes” auto-registration, hardware specifications retrieval of newly discovered bare metal machines, configuration and provisioning with custom operating system images, machines reset, cleaning for re-provisioning or end-of-life retirement.
